# Week 2

What I learned:

- The difference between rebuilding an image and restarting a container
  -- Rebuilding an image updates the application's underlying blueprint (including code and dependencies), creating a brand new environment. Restarting a container simply stops and launches the exact same, existing container environment without applying any code changes or updates --
- How to use Air for hot reloading Go applications during development

Problems faced:

- Changes made to main.go were not reflected in the running application.
- Difficulty understanding why Docker was not updating the application after code changes.
- Confusion between image rebuilding, container recreation, and volume synchronization.
- Incorrect volume mount path (/app. instead of /app) causing source files to be mounted in the wrong location.

How I solved them:

- Verified running containers and images with Docker commands.
- Checked the container's filesystem to confirm where source files were mounted.
- Identified the typo in the volume mount target (/app.).
  Corrected the mount path to /app
- Restarted the environment using docker compose down and docker compose up --build.
